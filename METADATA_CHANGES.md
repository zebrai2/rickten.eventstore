## Behavior

### When Client Provides CorrelationId
```csharp
new AppendEvent(event, new[] { 
    new AppendMetadata("CorrelationId", "abc-123") 
})
// Results in: EventMetadata("Client", "CorrelationId", "abc-123")
```

### When Client Doesn't Provide CorrelationId
```csharp
new AppendEvent(event)
// Results in: EventMetadata("System", "CorrelationId", <auto-generated-guid>)
```

### Reaction Propagation
```csharp
// Trigger event has: EventMetadata("System", "CorrelationId", guid-1)
var corrId = triggerEvent.Metadata.GetGuid("CorrelationId");

// Reaction provides it as AppendMetadata
new AppendEvent(reactionEvent, new[] { 
    new AppendMetadata("CorrelationId", corrId) 
})
// Results in: EventMetadata("Client", "CorrelationId", guid-1)
// Value preserved, but source changes to Client (because AppendMetadata is client-provided)
```

## Rationale

1. **No Source Preservation**: Source indicates who provided the metadata in this specific append operation, not historical origin
2. **Simplified Implementation**: No EventMetadata unwrapping needed
3. **Consistent Rules**: 
   - AppendMetadata → Client source
   - Auto-generated → System source
4. **Security**: Clients can't spoof System source since they only use AppendMetadata
